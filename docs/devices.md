UNIVAC-IX Hardware Integration: Adding Discovered Devices to Active Configurations
----------------------------------------------------------------------------------

This deployment guide walks you through migrating network assets discovered by the UNIVAC-IX Discovery Engine into your active runtime environments.

When the discovery scanner fingerprints infrastructure endpoints on your local or surrounding subnets, it updates two distinct locations:

-   `master_database.vcf`: The central cross-era directory containing standard contact formatting alongside injected `X-UNIVAC` metadata extensions.
-   `storage_pipeline/hardware_node_library/`: A library of structured `.json` manifest profiles documenting specific device capabilities and inferred industrial protocols.

* * * * *

🚀 Method 1: The Automated Runtime Sync (Zero-Configuration)
------------------------------------------------------------

If you are using the integrated core applications included in this repository, device activation is fully automated. The transmission engine natively processes the main `.vcf` file as its live routing directory.

1.  Open your terminal interface in the root workspace directory.
2.  Launch the master transmission script:

    ```
    python3 univac_transmitter_core.py

    ```

3.  Execution Mechanics: The engine automatically cycles through the entries inside `master_database.vcf`. It parses the validation blocks, automatically filters out any nodes marked `X-UNIVAC-NODE-STATUS:OFFLINE`, and dynamically maps all `ONLINE` nodes straight into the live memory transmission queue.
4.  The transmitter immediately builds standard UNIVAC 12-character fixed words for the discovered nodes and schedules outbound Baudot/ITA2 signaling streams without requiring manual script modifications.

* * * * *

📝 Method 2: Manual Hardware Mapping via JSON Manifests
-------------------------------------------------------

When a new device (like a Modbus or EtherNet/IP PLC) is discovered, the engine creates an deployment profile inside the localized node library. You can use these manifests to copy parameters directly into active terminal configs or industrial driver files.

1.  Navigate to your node library repository folder:

    ```
    cd storage_pipeline/hardware_node_library/

    ```

2.  Open the JSON profile matching your target device's IP layout (e.g., `node_profile_192_168_1_50.json`) with a text editor:

    ```
    {
        "ip_address": "192.168.1.50",
        "primary_interface_port": 502,
        "architecture_class": "Programmable Logic Controller (PLC)",
        "capabilities": ["Coil_Control", "Register_Telemetry", "Industrial_Automation"],
        "inferred_protocol": "Modbus/TCP",
        "last_interrogated": "2026-06-15 14:21:00"
    }

    ```

3.  Copy the `ip_address` and `primary_interface_port` values.
4.  Open your third-party industrial controller software or local driver map file, and paste the values into your active network scanner setup to link the physical PLC node.

* * * * *

🗂️ Method 3: Manual Integration via vCard Extraction
-----------------------------------------------------

For deployments where data must be shared with external software (like terminal emulators, contact managers, or physical router configurations), individual device definitions can be extracted straight from the master database.

1.  Open `master_database.vcf` in any text editor.
2.  Isolate the target hardware entry generated by the auto-recovery scan loop:

    ```
    BEGIN:VCARD
    VERSION:3.0
    FN:NODE-PROGRAMMABLE_LOGIC_CONTROLLER_(PLC)-50
    TEL;TYPE=IP,DIRECT:192.168.1.50
    X-UNIVAC-PLC-NODE:NET-50
    X-UNIVAC-NET-TYPE:Modbus/TCP
    X-UNIVAC-NODE-STATUS:ONLINE
    END:VCARD

    ```

3.  For External Terminal Emulators: Extract the `TEL;TYPE=IP,DIRECT:` parameter to establish raw TCP socket bridges to the field equipment, and evaluate the `X-UNIVAC-NET-TYPE:` metadata string to determine whether to launch the Modbus or EtherNet/IP software driver modules.
4.  For Modern Mail & Contact Infrastructure: You can copy this block, paste it into an isolated plain text file saved with a `.vcf` extension, and import it directly into an Android, iOS, or Microsoft Outlook client directory to interact with the device.
