## 1. System Topology & Data Flow

The system decouples high-overhead input/output network operations from the high-precision mathematical execution plant.

## 3. Subsystem Threading & Memory Isolation
To insulate the mathematical execution matrix from physical latency drops on network ports or broken copper wires, the architecture isolates memory structures via `threading.Lock()` primitives.

### A. Ingestion Handlers (Network Layer)
* **`ThreadedSerialPortListener`**: Runs inside isolated thread contexts performing blocking byte-level `readline()` captures from raw serial devices. Decoded text strings are instantaneously processed by the NMEA or Weapon parser subroutines.
* **`JsonTcpCommandListener`**: Spawns ephemeral background worker threads for each active remote workstation link. Raw text buffers are serialized, checked against configuration boundary gates, and pushed to the state variable stack.

### B. Synchronization Matrix (The Router)
The **`BridgeNetworkRouter`** acts as a thread-safe broker between variables. The core calculation loop executes at a constant 50Hz interval ($dt = 0.02$ seconds) using static cache snapshots pulled across the lock structures. The main calculation loop never halts or waits for an input/output network operation to complete.

## 4. Operational Control Law Coupling
The core plant takes live variables from the network router and steps them across four tightly coupled mathematical blocks:

1. **`NAVMOD_Subsystem`**: Transforms positioning metrics and handles crab-angle leeway adjustments.
2. **`HYDRO_Subsystem`**: Assesses restricted water boundaries, bank suction lift vectors, and squat parameters to dynamically limit speed capabilities.
3. **`WAVEMOD_Subsystem`**: Tracks forward-looking sensors and runs an Autoregressive forecasting model to predict propeller ventilation and stern swell levels several seconds ahead of impact.
4. **`MIMOMOD_Subsystem`**: Maps gyroscopic and weight-shift moments caused by active weapon tracking rings, modifying the active hull roll calculations so that the **Rudder Roll Stabilization (RRS)** algorithm pre-compensates for physical shifts in the vessel's center of gravity.

---

## 5. Interface Serialization Specifications
Output actuation commands generated by the core control plant are normalized and pushed through the serializer array to output structured, industrial NMEA text sentences:

* **Propulsion Wire Stream (`$PUNVCPRP`)**: Dictates requested engine power levels, speed limits, and structural load telemetry over RS-422 differential copper pairs.
* **Steering Wire Stream (`$PUNVCSTR`)**: Outputs high-frequency, notch-filtered corrective rudder commands to drive the hydraulic ram valves while filtering out destructive wave noise components.
